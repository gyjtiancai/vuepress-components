错误类型 ☹️

Error

Error 是基类型，其他错误类型继承该类型。因此，所有错误类型都共享相同的属性（所有错误对象上的方法都是这个默认类型定义的方法）。浏览器很少会抛出 Error 类型的错误，该类型主要用于开发者抛出自定义错误。

var err = new Error('出错了');
err.message // "出错了"
1
2

TIP

上面代码中，我们调用Error构造函数，生成一个实例对象err。Error构造函数接受一个参数，表示错误提示，可以从实例的message属性读到这个参数。抛出Error实例对象以后，整个程序就中断在发生错误的地方，不再往下执行。

InternalError

InternalError 类型的错误会在底层 JavaScript 引擎抛出异常时由浏览器抛出。例如，递归过多导 致了栈溢出。这个类型并不是代码中通常要处理的错误，如果真发生了这种错误，很可能代码哪里弄错 了或者有危险了。

EvalError

EvalError 类型的错误会在使用 eval()函数发生异常时抛出。基本上，只要不把 eval()当成函数调用就会报告该错误

//错误
new eval(); // 抛出 EvalError 
eval = foo; // 抛出 EvalError
//正确
eval("2+2")  // 4

RangeError

RangeError 错误会在数值越界时抛出。例如，定义数组时如果设置了并不支持的长度，如-20 或 Number.MAX_VALUE，就会报告该错误

// 1.数组长度为负数
new Array(-1)
//Uncaught RangeError: Invalid array length(无效的数组长度)

ReferenceError

ReferenceError 会在找不到对象时发生。（这就是著名的"object expected"浏览器错误的原 因。）这种错误经常是由访问不存在的变量而导致的

// 1.使用一个不存在的变量
aaa
//Uncaught ReferenceError: aaa is not defined(aaa没有定义)

// 2.将一个值分配给无法分配的对象(this不能赋值)
this = 1; 1 = 2
// ReferenceError: Invalid left-hand side in assignment(分配中的左侧无效)

TypeError

TypeError 在 JavaScript 中很常见，主要发生在变量不是预期类型，或者访问不存在的方法时。很 多原因可能导致这种错误，尤其是在使用类型特定的操作而变量类型不对时。

// 1.对非构造函数进行new命令   
new 'string'
//Uncaught TypeError: "string" is not a constructor("string"不是一个构造函数)
new 123
//Uncaught TypeError: 123 is not a constructor(123不是一个构造函数)
// 2.对象不存在
let obj = null
obj.name
//Uncaught TypeError: Cannot read property 'a' of null(无法读取null的属性'a')

URIError

最后一种错误类型是 URIError，只会在使用 encodeURI()或 decodeURI()但传入了格式错误的 URI 时发生。这个错误恐怕是 JavaScript 中难得一见的错误了，因为这两个函数非常稳健。

// 1. URI 相关函数的参数不正确
decodeURI('1*(&(&%')
//Uncaught URIError: URI malformed(URI格式错误)

不同的错误类型可用于为异常提供更多信息，以便实现适当的错误处理逻辑。在 try/catch 语句 的 catch 块中，可以使用 instanceof 操作符确定错误的类型

try {
    //dosomething
}catch(error) {
    if(error instanceof ReferenceError) {
        // 对象不存在
    }else if(error instanceof URIError) {
        //encodeURI()或 decodeURI()使用错误
    }else {
        //其他情况
    }
}

总结

以上这几种派生错误，连同原始的Error对象，都是构造函数。开发者可以使用它们，手动生成错误对象的实例。这些构造函数都接受一个函数，代表错误提示信息（message）

let error = new Error('出错了！');
let error_range = new RangeError('出错了，变量超出有效范围！');
let err_type = new TypeError('出错了，变量类型无效！');

error.message // "出错了！"
error_range.message // "出错了，变量超出有效范围！"
err_type.message // "出错了，变量类型无效！"
