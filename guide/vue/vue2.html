生命周期

话不多说 先上官方文档的模型示图

beforeCreate

vue 实例化之后 此时组件的data和methods以及页面DOM结构，都还没有初始化

created

组件的data和methods已经可以用了；但是页面还没有渲染出来；在这个生命周期函数中，我们经常会发起ajax请求

beforeMount

当模板在内存中编译完成，会立即执行实例创建阶段的第三个生命周期函数beforeMount，此时内存中的模板结构，还没有真正渲染到页面上；此时，页面上看不到真实的数据，用户看到的只是一个模板页面而已；

mounted

mounted是组件创建阶段最后的一个生命周期函数；此时，页面已经真正的渲染好了，用户可以看到真实的页面数据了；当这个生命周期函数执行完，组件就离开了创建阶段，进入到了运行中的阶段；如果大家使用到一些第三方的UI插件，而且这个插件还需要被初始化，那么，必须在mounted中来初始化插件

beforeUpdate

在执行beforeUpdate运行中的生命周期函数的时候，数据是最新的；但是页面上呈现的数据还是旧的（钩子函数debugger可直观调试）

updated

页面也完成了更新，此时，data数据是最新的，同时，页面上呈现的数据也只最新的

beforeDestroy

执行beforeDestroy的时候，组件即将被销毁，但是还没有真正开始销毁，此时组件还是正常可用的；data、methods等数据或方法，依旧可以被正常访问，但是路由已经切换

beforeDestroy和destroyed钩子

  • 触发场景(路由)

1.当前路由不使用缓存，离开当前路由会直接调用 beforeDestroy 和 destroyed
2.当前路由使用缓存，需要使用路由钩子函数主动的调用

beforeRouteLeave(to, from, next) {
  this.$destroy();
  next();
}

demo-js

export default {
  name: "",
  data() {
    return {
      msg: "hello",
      timer:null,
    };
  },
  components: {},
  computed: {

  },
  watch: {
    msg: {
      handler() {
        console.log("watch", this.msg);
        console.log("watch", this.func);
        console.log("watch", this.$refs["dom"]);
      },
    },
    immediate: true,
    deep: true,
  },
  beforeCreate() {
    /* vue 实例化之后 此时组件的data和methods以及页面DOM结构，都还没有初始化 */
    console.log("beforeCreate：", this.msg); //undefined
    console.log("beforeCreate：", this.func); //undefined
    console.log("beforeCreate：", this.$refs["dom"]); //undefined
  },
  created() {
    /* 组件的data和methods已经可以用了；但是页面还没有渲染出来；
    在这个生命周期函数中，我们经常会发起ajax请求 */
    console.log("created：", this.msg); //hello
    console.log("created：", this.func); //func(){console.log("func")}
    console.log("created：", this.$refs["dom"]); //undefined
    this.timer = setInterval(()=>{
      this.func()
    },1000)
  },
  beforeMount() {
    /* 模板在内存中编译完成，相关的 render 函数首次被调用;
    此时内存中的模板结构，还没有真正渲染到页面上 */
    console.log("beforeMount：", this.msg); //hello
    console.log("beforeMount：", this.func); //func(){console.log("func")}
    console.log("beforeMount：", this.$refs["dom"]); //undefined
  },
  mounted() {
    /* mounted是组件创建阶段最后的一个生命周期函数；
    此时，页面已经真正的渲染好，用户可以看到真实的页面数据了；
    当这个生命周期函数执行完，组件就离开了创建阶段，进入到了运行中的阶段；
    如果大家使用到一些第三方的UI插件，而且这个插件还需要被初始化
    那么，必须在mounted中来初始化插件 */
    console.log("mounted：", this.msg); //hello
    console.log("mounted：", this.func); //func(){console.log("func")}
    console.log("mounted：", this.$refs["dom"]); //<p class="msg">hello</p>
  },
  beforeUpdate() {
    /* 在执行beforeUpdate运行中的生命周期函数的时候，数据是最新的；
    但是页面上呈现的数据还是旧的 */
    console.log("beforeUpdate：", this.msg);//hi
    console.log("beforeUpdate：", this.func);//func(){console.log("func")}
    console.log("beforeUpdate：", this.$refs["dom"]);//<p class="msg">hi</p>
    //debugger  //debugger是为了查看页面上呈现的数据是新的还是旧的(答案是旧的 msg还是hello)
  },
  updated() {
    /* 页面也完成了更新，此时，data数据是最新的，同时，页面上呈现的数据也只最新的 */
    console.log("updated：", this.msg);//hi
    console.log("updated：", this.func);//func(){console.log("func")}
    console.log("updated：", this.$refs["dom"]);//<p class="msg">hi</p>
  },
  beforeDestroy() {
    /* 执行beforeDestroy的时候，组件即将被销毁，
    但是还没有真正开始销毁，此时组件还是正常可用的；
    data、methods等数据或方法，依旧可以被正常访问，但是路由已经切换 */
    console.log("beforeDestroy：", this.msg);//hi
    console.log("beforeDestroy：", this.func);//func(){console.log("func")}
    console.log("beforeDestroy：", this.$refs["dom"]);//<p class="msg">hi</p>
    //debugger //debugger是为了验证data、methods，dom还能使用 但是路由已经切换
    clearInterval(this.timer)
  },
  destroyed() {
    /* 实例销毁后调用。该钩子被调用后，对应 Vue 实例的所有指令都被解绑，
    所有的事件监听器被移除，所有的子实例也都被销毁 */
    console.log("destroyed：", this.msg);//hi
    console.log("destroyed：", this.func);//func(){console.log("func")}
    console.log("destroyed：", this.$refs["dom"]);//undefined
  },
  methods: {
    func() {
      console.log("func");
    },
  },
};
